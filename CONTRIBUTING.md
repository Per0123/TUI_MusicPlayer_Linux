# Contributing to TUI_MusicPlayer_Linux

Thank you for considering contributing to this project! Your help is much appreciated. Below are some guidelines to make contributing easier and more organized.

---

## Reporting Issues
If you encounter any bugs, crashes, or unexpected behavior, please open an issue with as much detail as possible, including:
- Steps to reproduce the issue
- Your operating system and environment
- Any error messages or screenshots

---

## Submitting Changes
If you want to contribute code or improvements:

1. Fork the repository.
2. Create a new branch:
   ```bash
   git checkout -b feature-branch
3. Make your changes and commit them with a clear message:
   ```bash
   git commit -am "Add new feature or fix issue"
5. Push your branch to your fork:
   git push origin feature-branch
6. Open a pull request to the main branch of this repository.

> **Tip:** Keep pull requests **focused** and **small** — it makes reviewing faster.

---

## Code Style
- Follow the existing code style and indentation.  
- Write clear and descriptive commit messages.  
- Keep functions and modules as simple and readable as possible.  

---

## Recognition
All contributors will be acknowledged in the [`THANKS.md`](./THANKS.md) file.  
Whether you submit code, report issues, help with testing, or suggest improvements, your contribution will be recognized and appreciated!

---

## Non-Code Contributions
Even if you don’t write code, you can still contribute by:

- Testing the application  
- Reporting bugs or unexpected behavior  
- Suggesting features or improvements  
- Helping with documentation  

All contributions will be recognized in the [`THANKS.md`](./THANKS.md) file.

---

## Thank You!
Even small contributions, feedback, or testing are appreciated.  
Your help makes **TUI_MusicPlayer_Linux** better for everyone!
